04

Iconography

The icon system maps to Bem.ai's architecture. Three visual layers represent three conceptual layers: what the system does (Functions), what it ingests (Inputs), and how it processes data (Operations).

Complete Icon System
All Layers — Functions (3D), Inputs (2D), Operations (Iconic) bem-final-idv-12.svg
Layer 1 — Functions 3D Shapes

Core processing functions rendered as three-dimensional geometric forms. These use the blue glow effect and represent the primary capabilities of the platform.

7 Function Icons — Full Set bem-final-idv-22.svg
Layer 2 — Inputs 2D Shapes

Data input types rendered as flat, two-dimensional forms. These represent the raw materials the platform ingests before processing.

6 Input Icons — Documents, Images, Spreadsheets, Structured Data, Audio, Video bem-final-idv-13.svg
Layer 3 — Operations Iconic

Data operations rendered as line icons. These represent the atomic data types the system works with internally.

5 Operation Icons — String, Number, Boolean, Object, Array bem-final-idv-14.svg
Usage
Blue or white stroke only1.5–2px stroke width. Blue for UI contexts, white for dark hero visuals.
