04

Iconography

The icon system maps to Bem.ai's architecture. Three visual layers represent three conceptual layers: what the system does (Functions), what it ingests (Inputs), and the atomic data types it works with (Data Types).

Complete Icon System
All Layers — Functions (3D), Inputs (2D), Data Types (Iconic) bem-final-idv-11.png SVG
Layer 1 — Functions 3D Shapes

Core processing functions rendered as three-dimensional isometric forms. These represent what the system does — the actions applied to data.

8 Function Icons — Extract, Transform, Analyze, Route, Split, Join, Payload Shaping, Enrich bem-final-idv-21.png
Layer 2 — Inputs 2D Shapes

Data input types rendered as flat, two-dimensional forms. These represent the raw materials the platform ingests before processing.

6 Input Icons — Documents, Images, Spreadsheets, Structured Data, Audio, Video bem-final-idv-12.png SVG
Layer 3 — Data Types Iconic

Data types rendered as line icons. These represent the atomic data types the system works with internally.

5 Data Type Icons — String, Number, Boolean, Object, Array bem-final-idv-13.png SVG
Simplified — Functions Flat / UI

Geometric stroke versions for small sizes, monochrome contexts, UI elements, and documentation. SVG, 64x64 viewBox, 2px stroke.
